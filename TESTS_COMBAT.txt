TESTS COMBAT SANDBOX — Phases 26 à 28

Couvre : Patch 1 (couche actions), Patch 2 (GDK), Patch 3 (reconnect timer),

PRÉ-REQUIS AVANT TOUT TEST
--------------------------------------------------------------------------------
1. Eclipse → Project → Clean → sélectionner le projet Sandbox → OK.
2. Vérifier qu'il n'y a aucune erreur de compilation rouge.
3. Relancer le serveur (org.dofus.Main).
4. Garder la console serveur visible pour lire les logs [SENT]/[RECV] et [FIGHT].

Chaque test précise :
  - ÉTAPES = ce qu'il faut faire dans le client
  - ATTENDU = comportement correct attendu
  - LOG = paquet à voir côté serveur pour valider
  - ÉCHEC = signe que le patch n'a pas marché

TEST 1 — LANCEMENT PvM + SPRITES VISIBLES (Patch 1 + 2)
================================================================================
ÉTAPES
  1. Se connecter avec un personnage niveau >= 1.
  2. Aller sur une map avec un groupe de monstres (ex: 7430, 7412).
  3. Cliquer sur le groupe → laisser le déplacement roleplay se terminer.

ATTENDU
  - Animation roleplay finie → écran combat charge.
  - Cases de placement (ronds rouges côté joueurs, bleus côté mobs) VISIBLES.
  - Sprites des joueurs et des mobs VISIBLES dans des cellules différentes.
  - Bouton "prêt" disponible.

LOG (ordre canonique)
  RECV GKK1
  SENT GM|-<groupId>
  SENT GJK2|0|1|0|30000|4          ← type 4 PvM
  SENT GP<places0>|<places1>|0
  SENT GM|+<player>
  SENT GM|+<mob1> ... GM|+<mobN>
  SENT GIC|<player>;<cell>|<mob>;<cell>|...
  SENT GDK                          ← NOUVEAU patch 2
  [FIGHT 1] placement started: ... vs group ... on map ...

ÉCHEC
  - Pas de GDK envoyé → sprites invisibles → patch 2 non appliqué.
  - Type GJK différent de 4 → patch 1 FIGHT_TYPE_PVM pas pris en compte.

TEST 5 — LANCER UN SORT DIRECT (Patch 1 + 5)
================================================================================
ÉTAPES
  1. À ton tour, sélectionner un sort de dégâts dans la barre.
  2. Cliquer sur un mob à portée.

ATTENDU

  - PA consommés (compteur baisse).
  - Vie du mob baisse (barre rouge réduite).

  SENT GA;402;<player>;<targetId>;0              ← si mort
  SENT GIC|...|GTM|...

ÉCHEC
  - GA;102 absent → PA pas consommés visuellement.
  - GA;100 avec ";dmg" au lieu de ",-dmg" → format ancien, patch 1 non appliqué.

TEST 8 — TOUR DES MONSTRES AVEC DÉLAIS (Patch 4)
================================================================================
ÉTAPES
  1. Passer ton tour rapidement (bouton Gt ou attendre).
  2. Observer les mobs jouer un par un.

ATTENDU
  - Chaque mob attend ~600 ms après GTS avant de bouger.
  - L'animation de déplacement se voit ENTIÈREMENT avant que les PM descendent.
  - Entre la fin du tour d'un mob et le début du suivant : ~300 ms perceptibles.
  - Aucun mob ne "se téléporte" ni "court sur place".

LOG (timestamps importants)
  SENT GTS<mob1>|30000
  (≥ 600 ms)
  SENT GA1;1;<mob1>;<path>
  (≥ steps × 300 + 400 ms = ~700-1300 ms)
  SENT GA;129;<mob1>;<mob1>,-<steps>
  SENT GA;100;<mob1>;<player>,-<damage>   ← si attaque
  (≥ 800 ms)
  SENT GTF<mob1>
  (≥ 300 ms)
  SENT GTS<mob2>|30000

ÉCHEC
  - GA1;1 et GA;129 dans la même milliseconde → délai animation pas appliqué.
  - GTF et GTS suivant dans la même ms → délai endTurn pas appliqué.
  - Patch 4 non compilé.

TEST 9 — DÉCO/RECO PENDANT COMBAT ACTIF (Patch 2 + 3)
================================================================================
ÉTAPES
  1. Lancer un combat, jouer 1-2 tours.
  2. Attendre que ce soit le tour d'un mob (timer du mob en cours).
  3. Fermer le client brutalement (Alt+F4 ou kill task).
  4. Reconnecter le même compte (même perso).

ATTENDU
  - Le client recharge directement dans le combat (pas écran noir, pas map old).
  - Sprites de tous les fighters visibles aux bonnes cellules.
  - Timeline visible avec ordre correct.
  - Le timer du mob courant affiche le TEMPS RESTANT (pas 30 000 ms plein).
  - À ton prochain tour, le timer est de nouveau 30 000 ms.

LOG (séquence reconnect)
  RECV GI
  SENT GJK2|...|4                         ← spectator=0, fight ACTIVE
  SENT GP<places>...
  SENT GM|+<all fighters>
  SENT GIC|<all positions>
  SENT GDK                                ← patch 2
  SENT GS
  SENT GTL|...
  SENT GTM|...                            ← état complet (vies actuelles)
  SENT GTS<currentMob>|<remainingMs>      ← patch 3 : temps RESTANT

ÉCHEC
  - GDK absent → écran noir ou sprites invisibles.
  - GTS avec 30000 alors que tour entamé → patch 3 non appliqué.
  - GTM absent → barres de vie remises à max (incorrect).

TEST 11 — VICTOIRE + TABLEAU DE FIN AVEC XP/KAMAS/DROPS (Patch 1)
================================================================================
ÉTAPES
  1. Combattre jusqu'à tuer TOUS les mobs.
  2. Observer le panneau de fin de combat.

ATTENDU
  - Panneau de résultat s'affiche automatiquement.
  - XP gagnée visible (valeur > 0 si mobs avaient XP).
  - Kamas gagnés visibles (valeur > 0).
  - Drops listés (si chance roll réussi) : nom_item × quantité.
  - Après fermeture du panneau, retour automatique sur la map roleplay.
  - Le groupe monstre ne réapparait PAS immédiatement (respawn scheduled).

LOG
  SENT GA;100;<player>;<lastMob>,-<damage>
  SENT GA;103;<player>;<lastMob>
  SENT GA;402;<player>;<lastMob>;0
  SENT GA;102;<player>;<player>,-<apCost>
  SENT GIC|...|GTM|...|<lastMob>;1     ← mob marqué dead
  SENT As<xpStuff>...                    ← stats mises à jour
  SENT Of+<kamas>                        ← kamas gagnés
  SENT GE<duration>|<senderId>|4|<result entries>
  SENT GV                                ← NOUVEAU patch 1 (signal sortie)
  [FIGHT N] ended winnerTeam=0 xp=X kamas=Y drops=Z

ÉCHEC
  - GV absent → client reste bloqué en combat, panneau pas finalisé.
  - GE absent → pas de panneau du tout.
  - XP/kamas à 0 dans le panneau alors qu'il y a des mobs morts → bug rewards.
  - GE avec |0| au lieu de |4| → FIGHT_TYPE_PVM pas mis à 4.
